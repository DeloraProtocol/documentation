Use this endpoint to read cached USD price snapshots for one token or a small batch of tokens.

## Request formats

For a single token, pass `chainId` and `token`:

```http
GET /v1/prices?chainId=8453&token=0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913
```

For multiple tokens, pass `tokens` as comma-separated `chainId:token` entries:

```http
GET /v1/prices?tokens=8453:0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913,1:0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48
```

## Response fields

Each item in `prices` includes:

| Field | Description |
| --- | --- |
| `chainId` | Chain ID for the resolved token price request. |
| `token` | Token address or symbol used for the resolved price request. |
| `priceUSD` | Cached USD price, when available. |
| `source` | Price source, when available. |
| `updatedAt` | Timestamp for the cached price, when available. |
| `stale` | Whether the cached value is outside Delora's freshness window. |
| `tier` | Price freshness tier: `hot` or `cold`. |
| `refreshing` | Whether a background refresh is scheduled or already running. |

## Additional Notes

Prices are returned immediately from Delora's cache. If a cached value is stale, the response marks it with `stale: true`; when a refresh is scheduled or already running, `refreshing` is also `true`.

<Note>
  `tokens` supports up to 50 requested entries per request.
</Note>
