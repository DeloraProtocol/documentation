This endpoint enables you to request a quote for transferring tokens, whether across chains or within the same chain. It returns detailed information, including an estimated output amount, an execution-ready `calldata` payload, and optional non-blocking `warnings`. The estimated output amount can be found in the `outputAmount` field.

**Base URL:** `https://api.delora.build/v1`

## Additional Notes

<Info>
  All Delora APIs don't require an API key by default. An API key is only necessary for accessing higher request limits.
</Info>

<Note>
  `senderAddress` is required. `receiverAddress` is optional only when the origin and destination chain types match; otherwise it must be provided explicitly.
</Note>

<Note>
  Address validation is chain-specific: `senderAddress` is validated against the origin chain type, and `receiverAddress` is validated against the destination chain type. Invalid or incompatible addresses return `400 Bad Request`.
</Note>

### Address Behavior

| Route type | `senderAddress` | `receiverAddress` |
| ---------- | --------------- | ----------------- |
| Same chain type (`EVM -> EVM`, `SVM -> SVM`) | Required | Optional. Defaults to `senderAddress` if omitted. |
| Different chain types (`EVM -> SVM`, `SVM -> EVM`) | Required | Required |

Example:

```http
GET /v1/quotes?originChainId=1&destinationChainId=42161&originCurrency=0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48&destinationCurrency=0xFF970A61A04b1cA14834A43f5dE4533eBDDB5CC8&senderAddress=0xYourSenderAddr&amount=1000000
```

This is valid because the route stays within the same chain type (`EVM -> EVM`), so `receiverAddress` can default to `senderAddress`.

```http
GET /v1/quotes?originChainId=1&destinationChainId=1000000001&originCurrency=0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48&destinationCurrency=EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v&senderAddress=0xYourSenderAddr&amount=1000000
```

This returns `400 Bad Request` because the route is `EVM -> SVM`, so `receiverAddress` must be provided explicitly.

### Adapter Filters

Use adapter filter parameters when you need to constrain route discovery to specific providers.

| Parameter | Behavior |
| --- | --- |
| `includeBridges` | Comma-separated bridge adapter allowlist for cross-chain route legs. |
| `includeExchanges` | Comma-separated exchange adapter allowlist for same-chain route legs. |
| `excludeBridges` | Comma-separated bridge adapter denylist. |
| `excludeExchanges` | Comma-separated exchange adapter denylist. |

Adapter values are the `key` values returned by `/v1/tools`.

<Note>
  Bridge filters accept bridge-capable adapters, and exchange filters accept exchange-capable adapters. Invalid adapter names or category mismatches return `400 Bad Request`.
</Note>

### Optional Response Fields

Depending on the selected route and backend configuration, the quote response can include additional metadata:

| Field | Behavior |
| --- | --- |
| `estimatedTimeSec` | Estimated bridge completion time. Returned for bridge routes when available. |
| `approvalAddress` | EVM approval spender address. Returned for EVM-origin routes. |
| `transactionSize` | Serialized Solana transaction size, returned when Solana calldata is available. |
| `bridgeScan` | Bridge tracking metadata, returned when the selected bridge adapter provides it. |
| `simulation` | Best-effort transaction simulation metadata, returned only when simulation is enabled. |
| `usd` | Cached USD price snapshots for the origin and destination quote tokens. |
