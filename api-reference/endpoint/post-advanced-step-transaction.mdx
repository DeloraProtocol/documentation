Use this endpoint to turn one selected advanced route step into executable transaction data.

## Request body

| Field | Type | Description |
| --- | --- | --- |
| `step` | `object` | A step object from `route.steps[]` returned by `/v1/advanced/routes`. |
| `context.senderAddress` | `string` | Wallet address that will sign/send the transaction. |
| `context.receiverAddress` | `string` | Final receiver address for the route. |
| `context.previousStepReceivedAmount` | `string` | Actual amount received from the previous step. Required for later steps that depend on prior output. |
| `context.amountOverride` | `string` | Optional manual amount override for the step. |
| `context.slippage` | `number` | Optional slippage override for the live step quote. |

## Headers

| Header | Description |
| --- | --- |
| `x-api-key` | Optional API key for higher rate limits and fee rules. |

## Response

The response is the populated step. It includes the original step fields plus `transactionRequest`.

`transactionRequest` contains the executable payload for the wallet:

- `to`
- `data`
- `value`
- optional `gas`
- optional Solana `transactionSize`

## Additional Notes

`/v1/advanced/stepTransaction` re-quotes the step at execution time and returns a populated step with `transactionRequest`. This is the transaction payload the wallet should sign and submit.

See [Simple vs advanced routing](/protocol/simple-vs-advanced-routing) for when to use `/v1/quotes` versus advanced routes.

<Note>
  Always call this endpoint immediately before executing a step. Do not treat the estimate returned by `/v1/advanced/routes` as final executable transaction data.
</Note>

For the first step, pass the selected route's first `steps[0]` object and the user's sender/receiver context.

For second and later steps, pass the real output amount observed from the previous step as `previousStepReceivedAmount`. This lets Delora hydrate the next step with live amounts instead of stale route estimates.
