The current widget package does not expose a locale, language, or translation API.

## Current behavior

- built-in widget UI strings are English only
- there is no `language` prop
- there is no translation resource override
- there is no built-in i18n provider or dictionary export

## What can still be customized

You can point the wallet consent footer to your own documents with:

- `config.termsUrl`
- `config.privacyPolicyUrl`

These links can be localized on the host side, but they do not translate the widget UI itself.
